# skill-builder

새 skill 을 만들고, 돌려보고, 함께 다듬는 skill.

## 무엇을 하나

당신이 만들고 싶은 것을 자연어로 말하면 — "매일 아침 어제 코딩 작업을 Notion 에 정리" 또는 "서울 날씨 알려줘" — 필요한 정보를 묻고 동작하는 skill 을 만듭니다.

만든 뒤에는 **여기서 끝이 아니라** 같이 다듬는 단계로 들어갑니다:

1. 만든 skill 한번 실행 (어떤 입력으로 돌릴지 같이 정함)
2. 결과 같이 보면서 "여기 너무 길다 / 이게 빠졌다 / 이건 다른 방식이 낫다" 같은 피드백 받음
3. 피드백대로 skill 다시 생성
4. 다시 실행 → 다시 피드백 → 만족할 때까지

매 iteration 마다 누적 회수와 비용을 알려줍니다 ("3 회 iteration, 총 $0.32 사용").

## 언제 쓰나

- 자동화하고 싶은 워크플로가 있는데 그런 skill 이 아직 없을 때
- 같은 작업을 반복해서 하고 있어 묶고 싶을 때

## 언제 안 쓰나 (지금은)

- 며칠 전에 만든 skill 다시 수정 — 같은 skill 이름으로 skill-builder 다시 부르면 "처음부터" 가 됨. 진짜 "기존 것 수정" 모드는 future skill-editor 의 일
- 여러 skill 묶어 한 번에 실행하는 workflow — 추후 orchestrator builder
- skill 동작 벤치마크 (정량 평가) — 추후 skill-tester

## 어떻게 작동하나 (대충)

1. **물어봄**: 무엇 만들지, 어떤 데이터 / 어떤 트리거 / 어떤 결과
2. **생성**: SKILL.md (당신의 의도) + zipsa-dist 두 파일 (envelope 등 zipsa 내부) 을 staging 에 씀
3. **검증**: validate 자동 돌림
4. **실행**: install 후 첫 run
5. **다듬기**: 결과 보고 변경 요청 받아 다시 생성. 만족할 때까지 loop
